Routing rules:

- Execute the child skill named in the job packet first. Use this matrix only to resolve ambiguity or repair a bad packet.
- During ideation, always perform at least one target-domain, near-neighbor, and far-neighbor discovery attempt, then actively screen candidates before idea generation. Use broad metadata-only discovery for the first search pass (`depth=deep`, `searchMode=deep`, LLM-augmented planning, citation/related-work expansion, high query/candidate caps); run `pre_idea_discovery_config_lint.py`; then require PaperNexus import_workflow completion/authoritative sync for selected import tasks, split-reading evidence, an innovation slot map, and a committed proposal graph session when the remote exposes `proposal_graph_session` before `EXPERIMENT_IDEA_POOL.json` is accepted. Treat target-domain methods as closest-prior pressure unless the novelty audit proves they are not present in the current field; top-tier main method ideas should cite near/far-neighbor mechanism transfer, external-domain transfer, or cross-lane recombination. If proposal graph support is unavailable or diagnosis-only, record the fallback boundary before using legacy `idea_catalyst`/`research_controller`. Explicit degraded speculative ideation requires user approval with claim limits.
- For every stage that can write `.autoreskill/user_view/`, maintain `user_view/innovation_story/` as prose-first story design. It must explain why the method is credible and novel through current-field pressure plus neighbor/cross-lane transfer; it must not replace evidence artifacts or collapse into a contribution list.
- Every PaperNexus discovery result must pass through screening before graph work: raw candidates -> `PAPER_SELECTION_SCORECARD.json` -> `GRAPH_IMPORT_PLAN.json` -> `IMPORT_WORKFLOW_STATUS.json` from `import_workflow` queue/status/wait -> PaperNexus material view or split-reading evidence. Do not use raw discovery rows directly as novelty, method, baseline, limitation, or citation evidence.
- After ideation, prefer targeted discovery over broad reruns: closest-prior closure at `idea_gate`, selected-idea import/material closure at `experiment_plan`, negative/contradictory evidence at `analysis`, novelty/citation/baseline objections at `review_pressure`, and unresolved related-work/citation placeholders at `writing` or `submission_ready`.
- If a legacy project already has `EXPERIMENT_IDEA_POOL.json` without canonical `ideation/PRE_IDEA_EVIDENCE_GATE.json`, run `legacy_pre_idea_reconcile.py` and route repair to pre-idea evidence expansion instead of treating the stage as complete. A misplaced gate such as `orchestrator/PRE_IDEA_EVIDENCE_GATE.json` must be rebuilt or relocated only after validating the required lane/split-reading artifacts.
- Keep `code` and `experiment` under `Coder` unless a job packet explicitly asks Researcher to reconcile scientific intent.
- Do not let a child skill advance `goal_state.json`. After artifacts are written, mark the job complete and run another tick.
- If a linter fails, update the job as failed or retry with the exact blocker instead of manually advancing the stage.
